## 2024-05-15 - Optimizing dateutil relativedelta incremental loops inside parent loops
**Learning:** In Python, particularly in financial apps computing daily/monthly histories, nested loops where a `while` loop incrementally adds `relativedelta(months=1)` inside a daily history loop (`for current_day in ...`) results in severe O(N*M) performance degradation. The `relativedelta` object is notoriously slow. Additionally, repeatedly adding months to an end-of-month date (e.g., Jan 31 + 1 month = Feb 28, then Feb 28 + 1 month = Mar 28) causes unintended permanent shifts away from the end-of-month target.
**Action:** Replace iterative month addition with O(1) mathematical approximations (e.g., calculating elapsed months upfront using `delta.years * 12 + delta.months`) combined with a strictly limited constant-time validation loop (e.g., testing `[0, 1, 2]` offsets).

## 2025-01-30 - Modal and Icon Button Accessibility
**Learning:** Found multiple instances of icon-only buttons (Edit, Delete, Close) lacking `aria-label` or using inaccessible text replacements like `&times;`. Modals specifically rely on `&times;` which is confusing for screen readers.
**Action:** When working on modals or tables with action buttons, always check for `aria-label`. Replace `&times;` with `XMarkIcon` for better visual and accessibility consistency.
## $(date +%Y-%m-%d) - Standardize Modal UI and Accessibility
**Learning:** Standard modals in this codebase rely on `.modal-overlay`, `.modal-content`, and `.modal-header` CSS classes to maintain a consistent design system, along with standard button utility classes (`btn btn-secondary`, `btn btn-danger`). Custom hardcoded tailwind classes should be avoided. Accessibility is ensured by adding `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, and `aria-describedby` to the `.modal-content` element.
**Action:** When creating or refactoring modals, always use the predefined modal wrapper and header classes, standard button classes, and include the necessary ARIA attributes to map the title and description to the modal container.

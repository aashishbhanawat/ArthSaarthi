# Functional Requirement: FR14.4 - Android Native Enablement

## 1. Objective
Provide a seamless, standalone mobile experience for ArthSaarthi by enabling the full application (Backend + Frontend) to run natively on Android devices.

## 2. Background
ArthSaarthi was originally designed as a web/desktop application. To reach mobile users without requiring a remote server, we use Chaquopy to run the Python backend on Android alongside a React/Capacitor frontend.

## 3. Requirements

### 3.1. Standalone Backend
- The Python backend MUST run as a self-contained service within the Android application process.
- It MUST use SQLite for data persistence, stored in the app's internal `filesDir`.
- It MUST handle its own lifecycle (start/stop) to conserve device resources.

### 3.2. Native Frontend Integration
- Use Capacitor to package the React frontend into an Android APK.
- The frontend MUST communicate with the local Python backend via a secure loopback interface (127.0.0.1).
- Port discovery: The backend MUST notify the frontend of its dynamically assigned port.

### 3.3. Build & CI/CD Integration
- The Android build process MUST be integrated into the main GitHub Actions workflows (`release.yml` for releases, `test-builds.yml` for testing).
- Provide automated signing for production builds using secure GitHub Secrets.

### 3.4. Local Logic Alignment
- Shared logic (e.g., seeding, snapshots) MUST be aligned between `desktop` and `android` modes.
- Authentication should be streamlined for a single-user local environment.

## 4. Verification
- Android-specific backend tests MUST pass (`app/tests/test_android_mode.py`).
- Manual verification of the signed APK on a physical Android device.
- Successful GitHub Action build artifacts for the `android` platform.
